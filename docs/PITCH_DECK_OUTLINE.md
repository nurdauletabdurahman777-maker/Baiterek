# Pitch deck outline

1. **Fragmented journey:** entrepreneurs repeat data across portals; the holding rebuilds every service.
2. **FlowOS:** one portal plus a governed low-code service operating system.
3. **Entrepreneur journey:** goal → deterministic route → Business Passport → dynamic application → single account.
4. **Live proof:** 500M rule changed, quality-checked and published without frontend code.
5. **Platform architecture:** registry, runtime, rules, calculations, documents, workflow, versions, gateway.
6. **Governance:** source/relevance, audit, version history, deterministic Quality Gate, human approval.
7. **AI with guardrails:** compiler/diff accelerate analysis; confidence/source/review; never auto-publish.
8. **Scale:** one runtime for two very different services, extensible to 70+ configurations.
9. **Impact metrics:** fewer visible fields, higher prefill, shorter completion, friction analytics.
10. **Roadmap:** production identity/RLS, real connectors, pilot subsidiary, catalog migration, holding rollout.
